Bugs Fixed

Fix
Lodestone teleports firing during walks
NavPath.resolve() was routing through lodestones on the Taverley bank–obelisk path. Fixed by passing Movement.DISABLE_TELEPORTS flag to every NavPath.resolve() call.
Fix
Bot walking away from bank on startup
On start, the bot would fire the pathfinder from inside the bank and walk outside before returning. Root cause was incorrect Taverley bank coordinates — the target was 11 tiles from the actual booth. Corrected bankX from 2886 to 2875. Also fixed bank scene object wait loop which was calling the pathfinder while the player was already standing at the booth.
Fix
Scroll counter always showing zero
RS3 scroll names include the ability in parentheses (e.g. Bull Ant scroll (Unburden)), so the exact ^Bull ant scroll$ pattern never matched. Removed the trailing $ anchor from scrollPattern and switched countBackpack from matches() to find().
Fix
Run energy thresholds never triggering
Varp 463 returns 0–10000 but thresholds were being compared as 0–100 percentages. Rest would never fire since raw energy (e.g. 7500) was always above the threshold (35). Added / 100 division in getRunEnergy(). Also corrected the MiniMenu.interact call to match the logged DoAction(COMPONENT, 1, -1, 96010254) signature exactly.
Fix
All PouchType shard costs and tertiary ingredients wrong
Nearly every entry in the enum had incorrect shard counts and many had wrong tertiary items — the original data appeared to use XP values instead of shard costs. Rebuilt the entire enum from RS3 wiki data. Also fixed charms on Giant ent (Gold), Iron titan (Crimson), and Steel titan (Crimson).
Fix
Bot not stopping when out of charms or tertiary items
When a charm or tertiary ran out mid-session, the bot would spam "Missing" logs and walk to the bank repeatedly. Now detects missing charm or tertiary immediately in both handleInfusing and handleBanking and transitions directly to OUT_OF_SUPPLIES.
Fix
Long delays after preset load and scroll conversion
Replaced fixed 2500–3500ms post-preset wait with delayUntil(supplies appear). Removed dead 1500–3000ms wait after scroll conversion completes. Replaced fixed obelisk-click delay with delayUntil(interface opens).

Features & Improvements

Feature
Scroll conversion support
Optional mode to convert all pouches to scrolls after each infuse batch. Tracks scrolls made and scrolls/hr in the Statistics panel. Counter correctly handles stackable scrolls and variable quantities per pouch type.
Feature
Smart startup — skip unnecessary walks
On start, the bot checks where the player is standing. Already at the obelisk with supplies → infuse immediately. Already at the bank → load preset immediately. Only walks if genuinely away from both locations.
Feature
No-progress watchdog (4 minutes)
If no pouches or scrolls are produced for 4 minutes while not banking or walking, the bot triggers OUT_OF_SUPPLIES. Prevents silent hangs.
Feature
Persistent settings
Selected pouch, location, logout on empty, and convert to scrolls are saved on change and restored on script load via ScriptConfig.
Improve
Human-like movement & timing
Per-session reactionOffset (50–250ms) varies base click speed between sessions. Per-trip tripOffsetX/Y shifts walk destinations slightly each trip. bumpActionCount() inserts 1.5–6s attention-drift pauses every 8–20 actions. waitForMovement() waits for player.isMoving() rather than fixed timers.
Improve
Run / rest management
Rests randomly between 30–50% energy, threshold re-rolled each time. Resumes only at 100% with a 1.5–4s human settle delay before moving. Uses the confirmed DoAction signature for the run orb.
